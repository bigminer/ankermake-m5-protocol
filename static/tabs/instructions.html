Print from your slicer

ankerctl exposes an OctoPrint-compatible upload API, so any slicer that can "send to an OctoPrint host" can slice and print straight to your M5C. Point it at this address:

{{ request_host }}:{{ request_port }} {{ macro.clipboard_button("#octoPrintHost") }}

{% if slicer_token_configured %} Enter the configured slicer API key. {% else %} Leave the API key blank only when the slicer runs on this computer. {% endif %}

OrcaSlicer
  1. On the Prepare screen, click the connection icon {{ macro.bi_icon("reception-4") }} next to the printer dropdown
  2. Set Host Type to OctoPrint
  3. Set Hostname, IP or URL to {{ request_host }}:{{ request_port }}
  4. {% if slicer_token_configured %} Enter the configured API Key {% else %} Leave API Key empty only for a local slicer {% endif %}
  5. Click Test to confirm the connection, then OK
  6. Slice your plate, then click Print plate
PrusaSlicer & SuperSlicer

The same setup works here — configure a Physical Printer and use Send and Print:

  1. Open Printer Settings and click the gear icon {{ macro.bi_icon("gear-fill") }} to add a physical printer
  2. Set Host Type to OctoPrint, Hostname to {{ request_host }}:{{ request_port }}{% if slicer_token_configured %}, then enter the configured API key{% else %}; leave the API key blank only for a local slicer{% endif %}
  3. Slice, then use the Send and Print action ({{ macro.bi_icon("caret-right-fill") }} next to Export)

Hint: Ctrl+Shift+G slices and opens the send-to-printer dialog.

{{ macro.bi_icon("info-circle-fill") }} The printer can't store jobs, so upload-only isn't supported — a print starts as soon as the transfer finishes. With the pre-print hook enabled, expect a few minutes of heating and auto-leveling before the first move.
